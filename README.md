# Supply Chain Optimization with Databricks — MMF + Chronos-2

This project demonstrates an end-to-end supply chain optimization pipeline that runs entirely on **Databricks Serverless** (including serverless GPU for the forecasting step). The pharma supply chain has 3 manufacturing plants delivering 30 product SKUs to 5 distribution centers, which in turn serve 30–60 wholesalers each.

The key difference from the upstream solution accelerator: **`02_Fine_Grained_Demand_Forecasting`** has been rewritten to use the [Many Model Forecasting (MMF)](https://github.com/databricks-industry-solutions/many-model-forecasting) framework with the **Chronos-2** foundation models from AutoGluon — zero-shot, no per-series training, ~30s for 900 series on a single A10 GPU.

## Pipeline

| Step | Notebook | Purpose | Compute |
|---|---|---|---|
| 1 | `01_Introduction_And_Setup` | Initialize catalog + schema, generate synthetic source data | Standard serverless |
| 2 | `02_Fine_Grained_Demand_Forecasting` | MMF + Chronos-2-Small / Chronos-2 dual-model forecast, pick the lower-SMAPE winner | **Serverless GPU (A10, Env v5)** |
| 3 | `03_Derive_Raw_Material_Demand` | BOM traversal via `networkx` to compute raw material requirements | Standard serverless |
| 4 | `04_Optimize_Transportation` | Per-product LP via `pulp` + `applyInPandas` to minimize shipping costs | Standard serverless |
| 5 | `05_Data_Analysis_&_Functions` | Create UC SQL functions (`product_from_raw`, `raw_from_product`, `revenue_risk`) for downstream agents | Standard serverless |

## Getting Started

1. **Add the repo to your Databricks workspace** via *Repos → Add Repo → `https://github.com/rohan-parikh-db/supply-chain-mmf.git`*.
2. **Run `01_Introduction_And_Setup`** on standard serverless. Set the two widgets:
   - `catalog_name`: an existing catalog you have write access to (default `main`)
   - `db_name`: schema name to create (default `supply_chain_db`)
3. **Open `02_Fine_Grained_Demand_Forecasting`**, set its compute via *Configuration → Compute*:
   - Accelerator: **A10**
   - Environment version: **5**
4. **Run notebooks 02 → 05** in order. Each is a standalone task — there is no `%run` cascade because notebook 02 needs GPU compute while 03/04/05 run on standard serverless.

## Serverless compatibility notes

The upstream `lara-openai/databricks-supply-chain` accelerator was written for classic Databricks Runtime (DBR). To run on serverless Spark Connect, this fork makes the following changes:

- `02_Fine_Grained_Demand_Forecasting`: replaced the single-node `statsmodels.ExponentialSmoothing` + pandas-UDF baseline with `mmf_sa.run_forecast(active_models=["Chronos2Small", "Chronos2"], accelerator="gpu", serverless=True)`. Models register into UC under `{catalog}.{db}.<model>_<use_case>`. Sets `HF_HUB_ENABLE_HF_TRANSFER=0` and installs `hf_transfer` so HF downloads work whether or not the runtime pre-sets the env var.
- `03_Derive_Raw_Material_Demand`: replaced `graphframes` (uses `DataFrame.sql_ctx`, removed in Spark Connect) with a `networkx` traversal on `bom.toPandas()`. The BOM has <100 nodes so single-node is fine.
- `04_Optimize_Transportation`: wrapped `spark.conf.set("spark.databricks.optimizer.adaptive.enabled", "false")` in try/except — not settable on serverless.
- `_resources/00-setup.py`: replaced `dbutils.notebook.entry_point.getDbutils().notebook().getContext().tags().apply('user')` with `spark.sql("select current_user()")`.
- `_resources/01-data-generator.py`: added `%pip install statsmodels matplotlib`; wrapped AQE-config set in try/except.
- `_resources/02-generate-supply.py`: replaced `df.rdd.flatMap(...).collect()` with `[r[col] for r in df.collect()]` — Spark Connect does not expose `.rdd`.
